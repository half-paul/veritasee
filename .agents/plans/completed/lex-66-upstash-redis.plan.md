# Plan: Provision Upstash Redis (region-pinned)

## Summary

Add a new `packages/redis` workspace package that wraps `@upstash/redis` behind a lazy-initialized Proxy client, mirroring the shape of `packages/db`. Surface required env vars (`UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`) via `requireEnv`, expose typed helpers, add a Vitest smoke test (SET / GET / EXPIRE) gated on env, and add `/api/health/redis` to `apps/web` mirroring the existing DB health route. Provisioning of EU + US Upstash databases is performed by the developer in the Upstash console; the code only consumes the URLs/tokens.

## User Story

As a developer
I want a thin Redis client wrapper backed by Upstash region-pinned databases
So that we can implement PRD §5.5 quotas and §5.1 proxy cache in later phases.

## Patterns to Follow

### Lazy Proxy client (mirror packages/db/src/client.ts)

```ts
// SOURCE: packages/db/src/client.ts:1-23
import { neon } from '@neondatabase/serverless';
import { drizzle } from 'drizzle-orm/neon-http';
import { schema } from './schema';
import { requireEnv } from './env';

type Db = ReturnType<typeof drizzle<typeof schema>>;
let cached: Db | undefined;

export function getDb(): Db {
  if (!cached) {
    const sql = neon(requireEnv('DATABASE_URL'));
    cached = drizzle(sql, { schema });
  }
  return cached;
}

export const db: Db = new Proxy({} as Db, {
  get(_, prop, receiver) {
    return Reflect.get(getDb() as object, prop, receiver);
  },
});
```

### Required-env helper (mirror packages/db/src/env.ts)

```ts
// SOURCE: packages/db/src/env.ts:1-4
export function requireEnv(name: string): string {
  const v = process.env[name];
  if (!v) throw new Error(`Missing required env var: ${name}`);
  return v;
}
```

## Risks

| Risk | Mitigation |
|------|------------|
| `@upstash/redis` REST client requires Node runtime, not Edge | Set `export const runtime = 'nodejs'` on health route (matches DB route). Document constraint in `client.ts` if any consumer might run on Edge. |
| Tests fail in CI without Upstash credentials | Use the same env-gated `describe.skip` pattern as `pgvector.test.ts`; tests skip cleanly when vars unset. |
| Two-region provisioning is operational, not code | Plan only documents env var shape for the primary region; multi-region routing is explicitly deferred to VS-022 / VS-067 per issue dependencies. |
| Module-level env validation would crash apps/web build | Use lazy Proxy + first-call validation (matches `packages/db` pattern), so builds succeed without secrets. |
| `pnpm-lock.yaml` churn | Run `pnpm install` once and commit the updated lockfile alongside source. |
