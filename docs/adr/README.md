# Architecture Decision Records

This folder records significant architectural decisions for Veritasee Override.

We use a **MADR-lite** format. Each ADR has:

- **Status** — Proposed | Accepted | Superseded
- **Context** — What forces are at play; why a decision is needed.
- **Decision** — What we chose, in plain language.
- **Consequences** — What becomes easier, harder, or constrained.
- **Alternatives** — Options considered and why they were not chosen.

File names are `NNNN-short-title.md`, numbered sequentially. Newer ADRs supersede older ones by reference, never by deletion.
